# 🍽️ Σύστημα Διαχείρισης Τραπεζιών & Παραγγελιών Εστιάσης

Ένα ολοκληρωμένο σύστημα διαχείρισης τραπεζιών, ψηφιακού καταλόγου για πελάτες και πίνακα ελέγχου (Admin) για την άμεση εκτέλεση παραγγελιών σε πραγματικό χρόνο.

---

## 📌 Αναλυτική Λίστα Λειτουργιών & Requirements

### 🏷️ 1. Διαχείριση Τραπεζιών & Πίνακας Ελέγχου Admin
* **Δυναμική Δημιουργία Τραπεζιών:** Προσθήκη και καταχώριση νέων τραπεζιών με προσαρμοσμένα ονόματα/αριθμούς (π.χ. `Τραπέζι 4` ή `VIP-1`).
* **Καθαρισμός Demo Δεδομένων:** Αφαίρεση όλων των στατικών demo τραπεζιών και σύνδεση με τη βάση δεδομένων.
* **Δημιουργία QR Code ανά Τραπέζι:**
  * Αυτόματη παραγωγή μοναδικού QR Code για κάθε τραπέζι που οδηγεί στον τοπικό διακομιστή (`http://<local-ip>:5000/menu?table=<table_id>`).
  * Αφαίρεση demo συνδέσμων και δυνατότητα προεπισκόπησης, λήψης (download) ή εκτύπωσης του QR Code απευθείας από το Admin UI.
* **Χρωματικός Διαχωρισμός Κατάστασης (Status) Τραπεζιών:**
  * 🟢 **Active / Ελεύθερο:** Πράσινο κουμπί - Τραπέζι διαθέσιμο για πελάτες.
  * 🔴 **Occupied / Κατειλημμένο:** Κόκκινο κουμπί - Υπάρχουν πελάτες στο τραπέζι.
  * 🟠 **Reserved / Κεκλεισμένο:** Κίτρινο/Πορτοκαλί κουμπί - Κρατημένο τραπέζι.
  * ⚪ **Disabled / Απενεργοποιημένο:** Γκρι/Σκούρο Κόκκινο - Μη διαθέσιμο τραπέζι.
* **Παύση Παραγγελιών (Pause Orders):** Όταν ο Admin ενεργοποιεί την παύση παραγγελιών, εμφανίζεται αμέσως ενημερωτικό μήνυμα/banner στην πλευρά του πελάτη ότι το κατάστημα δεν δέχεται παραγγελίες.

---

### 📱 2. Ψηφιακός Κατάλογος Πελάτη (Customer Menu)
* **Απλοποίηση & Καθαρισμός UI:**
  * Αφαίρεση του κουμπιού αλλαγής θέματος (Theme toggle) και του κουμπιού δημιουργίας QR Code.
  * Αφαίρεση όλων των εικονιδίων (icons) από τα κουμπιά για καθαρή, text-focused εμφάνιση.
* **Ενιαία Μπάρα Ενεργειών:** Τοποθέτηση του κουμπιού **Ανασκόπηση** δίπλα στα κουμπιά ενεργειών:
  * `[ Μετρητά ]`
  * `[ Σερβιτόρος ]`
  * `[ Ανασκόπηση ]`
* **Έλεγχος Κατάστασης Τραπεζιού σε Πραγματικό Χρόνο:** Αν ένα τραπέζι είναι `Disabled`, `Occupied` ή `Reserved`, κλειδώνουν οι επιλογές παραγγελίας και εμφανίζεται σχετικό ενημερωτικό banner.
* **Αποσύνδεση Καλαθιού & Ιστορικού Παραγγελίας:**
  * Πατώντας **Αποστολή**, τα προϊόντα στέλνονται στην κουζίνα και το καλάθι του πελάτη **αδειάζει αμέσως**.
  * Αποφυγή διπλότυπων προϊόντων ή επανενεργοποίησης ήδη απεσταλμένων αντικειμένων.
* **Ανασκόπηση & Πληρωμένα Προϊόντα (Demo Flow):** Τα προϊόντα που σημαίνονται ως πληρωμένα μέσα από το παράθυρο της Ανασκόπησης εξαφανίζονται αυτόματα από τη λίστα ανεξόφλητων και το υπόλοιπο αναπροσαρμόζεται.

---

### 📋 3. Πίνακας Παραγγελιών Admin (Admin Orders View)
* **Διάταξη Kanban 3 Στηλών:** Οργάνωση των παραγγελιών σε 3 κατακόρυφες στήλες:
  1. **Received / Λήψη:** Νέες εισερχόμενες παραγγελίες.
  2. **Preparing / Προετοιμασία:** Παραγγελίες που ετοιμάζονται στην κουζίνα.
  3. **Ready / Έτοιμες:** Παραγγελίες έτοιμες για σερβίρισμα.
* **Συμπαγείς Κάρτες (Compact Cards):** Μειωμένα περιθώρια (padding/margins) και βελτιστοποιημένη γραμματοσειρά ώστε να χωρούν πολλές κάρτες ταυτόχρονα στις 3 στήλες.
* **Έξυπνος Διαχωρισμός / Append Παραγγελιών:**
  * **Εκκρεμείς (Pending) Παραγγελίες:** Νέα προϊόντα από το ίδιο τραπέζι προστίθενται (append) στην υπάρχουσα ανοιχτή παραγγελία (π.χ. `#ORD-40138`).
  * **Παραγγελίες σε Preparing / Ready:** Αν η αρχική παραγγελία έχει περάσει σε στάδιο προετοιμασίας ή είναι έτοιμη, η νέα αποστολή από το ίδιο τραπέζι δημιουργεί **νεο Order ID** (π.χ. `#ORD-40139`).
* **Διαγραφή ανά Προϊόν (Itemized Delete):**
  * Προσθήκη ξεχωριστού κουμπιού διαγραφής (`Delete` / `X`) δίπλα σε κάθε προϊόν μέσα στην κάρτα παραγγελίας.
  * Η διαγραφή ενός προϊόντος ενημερώνει αυτόματα το συνολικό ποσό της παραγγελίας σε Admin και Πελάτη.
* **Χρωματιστά Κουμπιά Ενεργειών Παραγγελίας:**
  * 🟢 **Accept (Πράσινο):** Αποδοχή και μεταφορά/καταγραφή της παραγγελίας στο Activity Tab.
  * 🔴 **Cancel (Κόκκινο):** Ακύρωση παραγγελίας.
  * 🟣 **Delete (Μοβ):** Οριστική διαγραφή της κάρτας παραγγελίας.
* **Εντοπισμός Καθυστερημένης Παραγγελίας (Stalled Orders):**
  * Έλεγχος χρόνου **ανά τραπέζι**.
  * Αν περάσουν **πάνω από 20 λεπτά** από την προηγούμενη παραγγελία/δραστηριότητα του ίδιου τραπεζιού, η νέα παραγγελία επισημαίνεται αυτόματα με την ένδειξη **`STALLED`**.
* **Καρτέλα Δραστηριότητας (Activity Tab):** Αποθήκευση όλων των αποδεκτών (Accepted) παραγγελιών σε ιστορικό με στοιχεία όπως: Τραπέζι, Λίστα Προϊόντων, Συνολικό Ποσό, Order ID και Ώρα/Ημερομηνία.

## ⚡ Επικοινωνία σε Πραγματικό Χρόνο (WebSockets)
* Χρήση **Socket.IO** για αμφίδρομη συγχρονισμένη επικοινωνία μεταξύ Admin και Πελατών:
  * `order_created` / `order_updated`
  * `status_changed`
  * `pause_orders_toggled`
  * `item_deleted`

## 🏗️ Αρχιτεκτονική Συστήματος & Τεχνολογίες

* **Backend Framework:** Python (Flask / FastAPI) ή Node.js (Express)
* **Real-time Communication:** WebSockets (Socket.IO)
* **Database:** SQLite / PostgreSQL / MongoDB
* **Frontend:** HTML5, Modern CSS (Flexbox & CSS Grid), Plain JavaScript (ES6+) / React
* **QR Generation:** Native QR Code rendering library (π.χ. `qrcode.js` / `python-qrcode`)

---

## 🔄 WebSocket Events Reference

| Event Name | Direction | Περιγραφή |
|---|---|---|
| `order_created` | Client ➔ Admin | Αποστολή νέας παραγγελίας από το τραπέζι. |
| `order_updated` | Client ➔ Admin | Προσθήκη νέων προϊόντων σε υπάρχουσα `Pending` παραγγελία. |
| `item_deleted` | Admin ➔ Client | Ενημέρωση πελάτη μετά τη διαγραφή μεμονωμένου προϊόντος από τον Admin. |
| `status_changed` | Admin ➔ Client | Αλλαγή κατάστασης τραπεζιού (`Active`, `Occupied`, `Reserved`, `Disabled`). |
| `pause_orders_toggled` | Admin ➔ Client | Ενεργοποίηση/Απενεργοποίηση της παύσης παραγγελιών στο κατάστημα. |
| `order_status_update` | Admin ➔ Client | Αλλαγή στάδιου παραγγελίας (`Received`, `Preparing`, `Ready`). |

## 📡 REST API Endpoints Overview

### 1. Τραπέζια (Tables)
* `GET /api/tables` - Ανάκτηση όλων των εγγεγραμμένων τραπεζιών.
* `POST /api/tables` - Δημιουργία νέου τραπεζιού (Name/Number).
* `PUT /api/tables/:id/status` - Ενημέρωση κατάστασης τραπεζιού (`Free`, `Occupied`, `Reserved`, `Disabled`).
* `GET /api/tables/:id/qr` - Παραγωγή/λήψη QR Code με το local server URL.

### 2. Παραγγελίες (Orders)
* `GET /api/orders` - Ανάκτηση ενεργών παραγγελιών για τον πίνακα Admin.
* `POST /api/orders` - Δημιουργία ή ενημέρωση (append) παραγγελίας.
* `DELETE /api/orders/:order_id/items/:item_id` - Διαγραφή συγκεκριμένου προϊόντος από την παραγγελία.
* `POST /api/orders/:order_id/accept` - Αποδοχή παραγγελίας & αποθήκευση στο Activity Log.
* `POST /api/orders/:order_id/cancel` - Ακύρωση παραγγελίας.
* `DELETE /api/orders/:order_id` - Οριστική διαγραφή παραγγελίας.

### 3. Δραστηριότητα (Activity Log)
* `GET /api/activity` - Ανάκτηση ιστορικού αποδεκτών παραγγελιών.
* `POST /api/admin/pause-orders` - Ενεργοποίηση/Απενεργοποίηση γενικής παύσης παραγγελιών.
